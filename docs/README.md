## üîÑ Maintenance Guidelines

### Adding New Documentation

1. Identify correct category (roadmap, seo, i18n, etc.)
2. Follow naming convention (PREFIX_descriptor_SUFFIX.md)
3. Update this README.md with new file link
4. Add category-specific README if creating new category

### Archiving Old Documents

1. Move completed work to `docs/reports/completed/`
2. Move obsolete docs to `docs/archive/`
3. Update README to reflect changes
4. Preserve in git history (use `git mv`)

### Updating Roadmaps

1. Archive old version to `docs/roadmap/ROADMAP_*_V{N}.md`
2. Create new version as `ROADMAP_*_V{N+1}_CURRENT.md`
3. Update this README to point to CURRENT version
